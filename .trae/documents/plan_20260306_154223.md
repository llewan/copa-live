# Refactor to API-Football Exclusive Source & Smart Polling

## 1. Remove Football-Data.org Provider
*   **Action**: Completely remove `FootballDataOrgAdapter` and its dependencies.
*   **Goal**: Ensure `https://v3.football.api-sports.io/` is the **exclusive** data source for all match data (historical, scheduled, and live).
*   **Implementation**:
    *   Delete `server/services/providers/football-data/` directory.
    *   Update `footballService.ts` to remove `FootballDataOrgAdapter` import and usage.
    *   Refactor `getMatches`, `syncUpcomingSchedule`, and `syncLiveMatches` to use `ApiFootballComAdapter` only.

## 2. Database-First Strategy with On-Demand Sync
*   **Logic**:
    *   **Historical/Future Dates**: `getMatches(date)` will **always** return data directly from the database. No external API calls.
    *   **Today's Matches**:
        1.  Fetch matches from the database.
        2.  Check for **Active Matches**: Are there any matches with status `LIVE`, `HT`, `ET`, `PEN`, or `SCHEDULED` (within 20 mins)?
        3.  Check **Rate Limit**: Has `syncLiveMatches` run in the last **15 minutes**? (using `auditService`).
        4.  **Condition**: If (Active Matches Exist) AND (Time Since Last Sync > 15 mins):
            *   Trigger `apiFootball.getMatches(today)`.
            *   Update the database with new scores/statuses.
            *   Return the updated matches.
        5.  **Else**: Return the cached matches from the database immediately.

## 3. Daily Cron Job (Schedule Sync)
*   **Action**: Configure a single daily cron job.
*   **Schedule**: Runs once a day (e.g., 06:00 UTC).
*   **Task**: Calls `footballService.syncUpcomingSchedule(today, today + 7 days)`.
*   **Implementation**:
    *   Use `apiFootball.getMatchesRange` to fetch the schedule for the next week.
    *   Update the database with scheduled matches.
    *   Ensure `vercel.json` only has the `daily` cron configured.

## 4. Client-Side Smart Polling
*   **Action**: Optimize polling in `Dashboard.tsx`.
*   **Logic**:
    *   **Default**: Poll every **5 minutes** (300,000ms).
    *   **Optimization**: Check the status of matches returned.
        *   If **NO** matches are `LIVE` or `SCHEDULED` (within 2 hours), stop polling or increase interval to **1 hour**.
        *   If matches are `LIVE`, keep 5-minute polling.
    *   **Config**: Add `POLLING_INTERVAL_ACTIVE` (5m) and `POLLING_INTERVAL_INACTIVE` (1h) constants.

## 5. Monitoring & Logging

*   **Implementation**:

    *   Log "Cache Hit" vs "API Refresh" decisions in `footballService`.
