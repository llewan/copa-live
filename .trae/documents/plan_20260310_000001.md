Activaré modo plan para ejecutar una auditoría completa de consistencia de diseño, sin modificar código de producto hasta tu aprobación.

### 1. Definir marco de auditoría y checklist unificado
- Construiré una matriz de evaluación por categorías: espaciado, tipografía, color, alineación, sombras, bordes, iconografía y patrones de interacción.
- Tomaré como baseline las fuentes reales del proyecto (`tailwind.config.js`, `index.css`, componentes y páginas enrutadas en `App.tsx`) para inferir las guías de estilo vigentes.
- Estableceré criterios de severidad (crítico, medio, menor) y reglas de aceptación visual por componente.

### 2. Levantar evidencia visual en todo el sitio
- Recorreré todas las rutas activas (`/`, `/match/:id`, `/login`, `/register`, `/preferences`) y estados relevantes (carga, vacío, interacción, responsive).
- Capturaré evidencia con screenshots comparables en desktop, tablet y móvil para detectar inconsistencias entre secciones y dispositivos.
- Documentaré contexto exacto de cada hallazgo (pantalla, componente, estado y viewport).

### 3. Detectar y catalogar inconsistencias
- Registraré cada inconsistencia con descripción objetiva: qué rompe, dónde aparece, impacto UX y patrón esperado.
- Agruparé por tipo (ej. tipografía, spacing, color) y por alcance (local, transversal).
- Verificaré coherencia entre componentes repetidos (cards, botones, headers, formularios, badges, estados de partido).

### 4. Validar patrones de interacción y comportamiento visual
- Revisaré hover, focus, active, disabled, feedback de acciones, jerarquía visual y consistencia de affordances.
- Contrastaré accesibilidad visual básica (contraste percibido, legibilidad, tamaños interactivos) para detectar desalineaciones con buenas prácticas.
- Confirmaré si la iconografía mantiene estilo, tamaño, peso visual y alineación con texto.

### 5. Entregar informe detallado con recomendaciones accionables
- Prepararé un informe estructurado con tabla de hallazgos: ID, severidad, categoría, evidencia (captura), causa probable y recomendación específica.
- Incluiré propuestas concretas de corrección (tokens, clases utilitarias, reglas de layout o componentes base) para cerrar cada inconsistencia.
- Añadiré un resumen ejecutivo con prioridades de implementación para correcciones rápidas vs. correcciones sistémicas.

### 6. Verificación final de coherencia global
- Haré una segunda pasada de control para asegurar cobertura completa de páginas y componentes clave.
- Confirmaré trazabilidad entre hallazgo y captura para que la revisión sea auditable.
- Entregaré el informe final listo para ejecución técnica en una fase posterior.

Si apruebas este plan, ejecuto la auditoría completa y te entrego el informe con capturas y recomendaciones específicas.
